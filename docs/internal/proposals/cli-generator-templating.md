# Proposal: Template-Driven CLI Generation

**Status:** Draft
**Date:** 2026-03-12
**Context:** Review of `rh-trex-ai/scripts/cli-generator/PLAN.md`

---

## Summary

The PLAN.md proposal from rh-trex-ai is correct in its goal — the CLI should be regenerable from the OpenAPI spec so adding a new Kind requires no CLI code change — but it proposes the wrong architecture for the generated code. This proposal accepts the generator-driven approach and the per-resource command structure, but rejects the removal of the ambient-sdk dependency and the raw-HTTP template model. It also adds several missing engineering concerns the plan does not address.

---

## What the Plan Gets Right

### 1. Generator-driven, spec-first CLI

The core idea is correct. Every CRUD Kind added to the API server should produce CLI commands automatically, the same way `make generate-sdk` produces SDK types. The current CLI (`cmd/get/cmd.go`) is a growing monolithic switch statement — every new Kind requires a manual edit to `get`, `create`, `delete`, and `describe`. This is the exact class of problem code generators solve.

### 2. Per-resource command tree structure

`acpctl session list` / `acpctl session get <id>` is better UX than `acpctl get session <id>`. Per-resource subcommand trees are the modern CLI standard (see: `gh`, `aws`, `gcloud`, `oc`). The plan is right to move in this direction. The monolithic per-verb dispatch (`get/cmd.go`, `create/cmd.go`) is a dead end as the resource count grows.

### 3. New templates: delete, describe, patch, testhelper, Makefile

These are all missing and all necessary. The current CLI has no `delete` with interactive confirmation, no `describe` (JSON dump), no `patch` (partial update), and no generated integration test infrastructure. Adding these templates is the right Phase 1 investment.

### 4. Hand-authored exceptions are the right concept

Not every command is derivable from a CRUD schema. `session messages --follow` (SSE stream), `whoami` (JWT introspection), `project set` (DNS label validation), `start` / `stop` (lifecycle actions) — these are genuinely ambient-specific business logic. The plan correctly identifies that these stay hand-authored.

## What the Plan Gets Wrong

### 1. Removing the ambient-sdk dependency

**This is the plan's central mistake.** The current `pkg/connection/connection.go` wraps `sdkclient.NewClient()` from the Go SDK. The plan's `connection.go.tmpl` replaces this with a raw `net/http` builder (`NewConnection().Config(cfg).Build()`) that speaks plain JSON.

**The consequences:**

| | SDK-backed (current) | Raw HTTP (proposed) |
|---|---|---|
| Type safety | Compile-time checked | None — raw `[]byte` |
| Table columns | From typed struct fields | From JSON key walk (fragile) |
| Request validation | In SDK | Duplicated or absent |
| API error handling | In SDK | Duplicated in every template |
| TLS/auth | In SDK | Duplicated in template |
| Drift risk | SDK and CLI stay in sync | Two independent clients can diverge |

The SDK exists precisely so callers don't implement their own HTTP layer. Removing it from the CLI defeats the purpose. When the API server changes a field name, the SDK type changes and the CLI breaks at compile time — that's the contract. Raw HTTP silently swallows those changes until runtime.

**The correct model:** The CLI generator templates should import from `ambient-sdk/go-sdk/client` and `ambient-sdk/go-sdk/types`. The generated `list_resource.go.tmpl` should call `client.Sessions().List(ctx, opts)` not `conn.List(urls.SessionsPath, page, size, ...)`. The generator needs a `--sdk-module` flag to parameterize the import path.

### 2. `table.WriteRawObject(item)` is unworkable for structured output

The `list_resource.go.tmpl` calls `table.WriteRawObject(item)` where `item` is `json.RawMessage`. To render a row, the table printer must walk the raw JSON and extract fields by name at runtime. This means:

- Column order is not guaranteed
- Type-specific formatting (age display, phase coloring, truncation) is impossible
- The `DefaultColumns` string from the generator becomes a fragile hint, not a contract

The current `get/cmd.go` renders typed columns (`s.ID`, `s.Phase`, `s.LlmModel`, `formatAge(time.Since(*s.CreatedAt))`) because it has SDK types. This is strictly better. The templates should be written against SDK types, with per-resource column definitions in the template itself.

### 3. No regeneration protection mechanism

The plan lists "what stays hand-authored forever" as a table in a markdown document. This is not enforceable. When someone runs `make generate-cli` six months from now, nothing prevents the generator from overwriting `cmd/session/messages.go`.

The SDK generator solves this with a `// Code generated by ambient-sdk-generator` header — editors and humans know not to edit these files. But it doesn't solve the inverse: protecting hand-authored files from being targeted by the generator.

Needed: a `.nogenerate` marker file (or directory-level exclusion list in the generator) so `cmd/session/` is never touched by `make generate-cli` even if a `Session` schema exists in the OpenAPI spec.

### 4. The patch command is underspecified

"Per-field flags, partial JSON PATCH" is three words for a genuinely hard problem. When `--name foo` is passed, `name` goes into the PATCH body. When `--name` is not passed, `name` is omitted (not cleared). But what if the user explicitly wants to clear a field? You need `--clear-name` or `--name ""` semantics, and the template must generate the right nil-vs-omit handling for each field type.

The PatchRequest schemas in the OpenAPI spec define which fields are patchable. The generator already parses these. But the template must generate code that builds a `map[string]any` (or typed patch struct) only for flags that were explicitly set — not all flags that have a non-zero value. Go's `cobra` doesn't distinguish "flag not passed" from "flag passed as zero value" without `Changed()` checks.

Required in the patch template:

```go
patch := map[string]any{}
if cmd.Flags().Changed("name") {
    patch["name"] = args.name
}
if cmd.Flags().Changed("description") {
    patch["description"] = args.description
}
```

This is generateable but must be explicit in the template design.

### 5. No spec SHA guard

The SDK generator stamps every generated file with:
```
// Spec SHA256: 9fc1cbdb0aa1a90c2d918eacbe4ed297f7ab6a3916b473be1bd5b82344f28f4d
// Generated: 2026-03-12T10:28:42Z
```

This lets the generator detect stale output and skip unnecessary regeneration. The CLI generator has no equivalent. Without it, `make generate-cli` is a blunt instrument — it always overwrites, even when nothing changed, which produces noisy diffs and makes code review harder.

## Proposed Architecture

### Templates call SDK, not raw HTTP

The generator gets a new flag: `--sdk-module github.com/ambient-code/platform/components/ambient-sdk/go-sdk`

Generated `list_resource.go.tmpl`:

```go
package {{.Resource.PluralLower}}

import (
    "context"
    "fmt"
    "os"
    "time"

    "github.com/spf13/cobra"
    sdkclient "{{.SDKModule}}/client"
    sdktypes "{{.SDKModule}}/types"
    "{{.Module}}/pkg/config"
    "{{.Module}}/pkg/connection"
    "{{.Module}}/pkg/output"
)

var Cmd = &cobra.Command{ ... }

func run(cmd *cobra.Command, argv []string) error {
    client, err := connection.NewClientFromConfig()
    ...
    opts := sdktypes.NewListOptions().Size(args.limit).Build()
    list, err := client.{{.Resource.Plural}}().List(ctx, opts)
    ...
}
```

The `connection.go.tmpl` becomes a thin wrapper around `sdkclient.NewClient()`, matching the current `connection.go` exactly. The TLS skip-verify TODO in the current file gets implemented as part of this work.

### Per-resource directory, with `.nogenerate` protection

```
cmd/acpctl/
  sessions/           # GENERATED — sessions/list.go, get.go, create.go, delete.go, describe.go, patch.go
  agents/             # GENERATED
  projects/           # GENERATED
  roles/              # GENERATED
  ...
  session/            # HAND-AUTHORED — .nogenerate present; messages.go, send.go, ag-ui.go
  whoami/             # HAND-AUTHORED
  project/            # HAND-AUTHORED — context management
  start/              # HAND-AUTHORED
  stop/               # HAND-AUTHORED
```

Note the naming convention: plural (`sessions/`) for generated per-resource CRUD, singular (`session/`) for hand-authored subresource commands. The generator skips any directory containing `.nogenerate`.

### Spec SHA in generated file headers

```go
// Code generated by cli-generator from openapi.yaml — DO NOT EDIT.
// Source: ../../ambient-api-server/openapi/openapi.yaml
// Spec SHA256: 9fc1cbdb0aa1a90c2d918eacbe4ed297f7ab6a3916b473be1bd5b82344f28f4d
// Generated: 2026-03-12T10:28:42Z
```

Generator skips a file if its SHA comment matches the current spec hash.

### Patch template with explicit `Changed()` checks

Generated per patchable field from the `*PatchRequest` schema, using `cmd.Flags().Changed("field-name")` to distinguish set vs omitted.

### Integration into the agent-build workflow

`make generate-cli` joins the workflow after `make generate-sdk`:

```
scripts/generator.go → openapi.yaml → make generate → make generate-sdk → make generate-cli → CLI commands appear
```

No CLI code changes needed for a new Kind — only hand-authored extras (streaming, watch, lifecycle) require manual work.

## Revised Phase Plan

### Phase 1 — Generator enhancements (rh-trex-ai)

1. Add `--sdk-module` flag to `cli-generator/main.go`
2. Rewrite `connection.go.tmpl` to wrap `sdkclient.NewClient()`, add TLS skip-verify
3. Rewrite `list_resource.go.tmpl` and `get_resource.go.tmpl` to use typed SDK calls
4. Add `delete.go.tmpl` — interactive `[y/N]` + `--yes` flag, calls `client.Xs().Delete(ctx, id)`
5. Add `describe.go.tmpl` — always JSON, calls `client.Xs().Get(ctx, id)`, pretty-prints
6. Add `patch.go.tmpl` — per-field flags from `*PatchRequest` schema, `Changed()` guards
7. Add spec SHA stamp to all generated file headers
8. Add `.nogenerate` directory exclusion logic to generator
9. Add `Makefile.tmpl` with `generate-cli`, `build`, `install`, `test` targets
10. Add `internal/testhelper/testhelper.go.tmpl`
11. Update `pkg/config.go.tmpl` — `Armed()`, `Disarm()`, `GetProject()`, `GetToken()`, `GetAPIUrl()`
12. Update `cmd/main.go.tmpl` — `PersistentPreRunE`, `--insecure-skip-tls-verify`, `Version:`

### Phase 2 — Migrate ambient-cli (ambient-cli)

1. Run `make generate-cli` against ambient OpenAPI spec
2. Replace monolithic `cmd/get/cmd.go`, `cmd/create/cmd.go`, `cmd/delete/cmd.go`, `cmd/describe/cmd.go` with generated per-resource trees
3. Add `.nogenerate` to `cmd/session/`, `cmd/whoami/`, `cmd/project/`, `cmd/start/`, `cmd/stop/`
4. Register generated resource commands in `cmd/acpctl/main.go`
5. Keep SDK dependency — do not remove
6. Implement TLS skip-verify in `connection.go` (currently a TODO)

### Phase 3 — Verification

```bash
make generate-cli
go build ./...
go test ./...
acpctl get sessions
acpctl sessions list
acpctl session messages <id>    # hand-authored still works
```

## Open Questions

| Question | Recommendation |
|---|---|
| How does `acpctl sessions list` coexist with `acpctl get sessions` during migration? | Keep `get/cmd.go` as a compatibility shim pointing to the same handlers until both are proven; then remove in a follow-up PR |
| Watch mode (`acpctl get sessions -w`) — where does it live? | Hand-authored in `cmd/session/watch.go` with `.nogenerate` on the directory |
| Should `acpctl session messages` register under `cmd/sessions/` or `cmd/session/`? | `cmd/session/` (singular, hand-authored). `cmd/sessions/` is the generated CRUD tree. |
| Can the patch template handle `time.Time` and nullable fields cleanly? | Yes — field type from OpenAPI determines the flag type; nullable fields get `--clear-<field>` as a boolean companion |

## What This Does Not Change

- The API server plugin generator (`scripts/generator.go`) — unchanged
- The SDK generator (`ambient-sdk/generator/`) — unchanged; it now auto-discovers Kinds
- The `ambient-data-model.md` design-first requirement — unchanged
- `make generate-sdk` — runs before `make generate-cli` in the workflow
- Hand-authored commands: `session messages`, `session send`, `session ag-ui`, `whoami`, `project`, `start`, `stop`
